# 07 Computação Serverless

Este módulo cobre execução de aplicações sem gerenciar servidores, integração orientada a eventos e orquestração.

## Objetivos de prova
- Entender o modelo de cobrança por uso em serverless.
- Diferenciar Lambda, API Gateway, Fargate e Step Functions.
- Identificar EventBridge como barramento de eventos.

## Ordem recomendada
1. AWS Lambda
2. API Gateway
3. AWS Fargate
4. Amazon ECS e EKS
5. AWS Step Functions
6. Amazon EventBridge

## Dica de memorização
- Função: Lambda
- Entrada HTTP: API Gateway
- Container sem servidor: Fargate
- Fluxo: Step Functions
- Evento: EventBridge
