# Glossário AWS A-Z

Este glossário resume termos-chave da trilha CLF-C02 para revisão rápida antes de simulados e prova.

## Como usar
- Revise os termos do domínio que você mais erra nos simulados.
- Priorize conceitos de segurança, custos e serviços fundamentais.
- Retorne ao módulo teórico correspondente quando necessário.

## A
- Auto Scaling: ajusta automaticamente capacidade computacional conforme demanda.
- AWS Organizations: gestão centralizada de múltiplas contas AWS.

## B
- Billing: área de faturamento e custos da conta.
- Budgets: define orçamento e alertas de custo/uso.

## C
- CloudFormation: infraestrutura como código por templates.
- CloudFront: CDN para entrega de conteúdo com baixa latência.
- CloudTrail: trilha de auditoria de chamadas de API.
- CloudWatch: métricas, logs, alarmes e eventos operacionais.
- Compliance: aderência a normas, políticas internas e requisitos regulatórios.

## D
- DynamoDB: banco NoSQL gerenciado de alta escala.

## E
- EC2: computação virtual sob demanda.
- EBS: armazenamento em bloco para EC2.
- EFS: filesystem compartilhado e elástico.
- ELB: balanceamento de carga (ALB/NLB/GWLB).
- Encryption at Rest: criptografia de dados armazenados.
- Encryption in Transit: criptografia de dados em tráfego de rede.

## F
- Fargate: execução serverless para containers no ECS/EKS.

## I
- IAM: gerenciamento de identidades, permissões e acesso.
- IAM Role: identidade assumível por serviço/usuário para permissões temporárias.

## K
- KMS: gerenciamento de chaves para criptografia.

## L
- Lambda: execução de código orientada a eventos, sem servidor.

## M
- Multi-AZ: implantação em múltiplas zonas para alta disponibilidade.
- MFA: autenticação multifator para reforçar segurança de acesso.

## R
- RDS: banco relacional gerenciado.
- Rekognition: análise de imagens e vídeos por IA.
- Route 53: DNS gerenciado com roteamento de tráfego.
- Resilience: capacidade de recuperar de falhas mantendo operação.

## S
- S3: armazenamento de objetos com múltiplas classes.
- Savings Plans: compromisso de uso para reduzir custo.
- Security Group: firewall virtual stateful para recursos em VPC.
- SNS: pub/sub para notificações e fan-out.
- SQS: fila de mensagens para desacoplamento.
- Shared Responsibility Model: divisão de responsabilidades de segurança entre AWS e cliente.
- SLA: acordo de nível de serviço com disponibilidade prevista.

## V
- VPC: rede virtual isolada na AWS.

## W
- Well-Architected Framework: boas práticas de arquitetura em seis pilares.
- WAF: firewall de aplicação web para proteção contra ataques comuns.
