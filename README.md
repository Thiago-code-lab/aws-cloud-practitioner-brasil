## 🎯 Sobre este Repositório

Este repositório combina conteúdo em Português do Brasil com foco no que realmente cai na prova CLF-C02: fundamentos de nuvem, segurança, principais serviços AWS e noções de custo/suporte.

O material foi organizado para funcionar bem em estudo progressivo: você aprende a base, evolui por domínio, revisa com questões e consolida com simulados.

## ⚠️ Armadilhas comuns na prova

| Confusão frequente | Quando usar cada opção | Armadilha típica de prova |
|---|---|---|
| Shared Responsibility Model | AWS protege a infraestrutura da nuvem; o cliente protege dados, identidade e configurações. | Assumir que a AWS configura IAM e políticas da conta do cliente. |
| Security Group vs NACL | SG: stateful e no nível da instância. NACL: stateless e no nível da subnet. | Marcar SG como stateless ou usar NACL para regra de aplicação. |
| S3 classes de armazenamento | Standard para acesso frequente; IA/One Zone-IA para acesso menos frequente; Glacier para arquivamento. | Escolher classe barata sem considerar tempo de recuperação e retenção mínima. |
| Multi-AZ vs Read Replica | Multi-AZ melhora disponibilidade/failover; Read Replica escala leitura. | Achar que Multi-AZ aumenta performance de leitura. |
| On-Demand vs Reserved vs Spot | On-Demand para flexibilidade, Reserved para previsibilidade, Spot para cargas tolerantes a interrupção. | Usar Spot em carga crítica sem estratégia de fallback. |
